# Data Dictionary

| Table | Grain | Purpose |
|---|---|---|
| `campus_cohorts.csv` | Campus cohort | Defines addressable market, student segment, region, product fit, partner fit, trust gap, cost index, and operating owner. |
| `weekly_funnel_metrics.csv` | Week x campus x channel | Tracks impressions, leads, applications, KYC flow, approvals, funded accounts, activated users, spend, support load, and rework. |
| `growth_experiments.csv` | Experiment | Scores potential growth tests by expected activations, spend, CAC, impact, confidence, ease, risk, decision, and owner. |
| `product_readiness.csv` | Readiness issue | Identifies onboarding, data, compliance, support, risk, and RevOps blockers by affected funnel stage and user impact. |
| `stakeholder_actions.csv` | Sprint action | Converts analysis into owner-assigned sprint actions with expected lift, effort, dependency, status, and workstream. |
| `analysis/outputs/cohort_priority_queue.csv` | Campus cohort | Ranked queue for where to focus the next growth sprint. |
| `analysis/outputs/experiment_backlog.csv` | Experiment | Sorted experiment backlog for activation tests. |
| `analysis/outputs/readiness_queue.csv` | Readiness issue | Ranked operational blockers before scaling acquisition. |
| `analysis/outputs/stakeholder_action_plan.csv` | Sprint action | Stakeholder-ready action plan for the nine-week sprint. |
| `analysis/outputs/app_payload.json` | App payload | JSON payload used by the interactive static workbench. |
